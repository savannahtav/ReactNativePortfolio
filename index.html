TODO App

The purpose of this project was to create a TODO application using React Native. The application allows users to manage their tasks effectively by providing features such as task listing, task completion, and task addition, as well as having the ability to login or register for an account.

The specific skills and technologies I utilized were React Native for building cross-platform mobile applications, AsyncStorage for local data storage, and various components from the React Native Elements library for UI elements.

One challenge I faced during development was managing asynchronous operations, such as fetching data from AsyncStorage and handling user interactions, in a synchronous and predictable manner. Asynchronous operations can introduce complexities, especially when dealing with state management and ensuring data consistency.

To overcome this challenge, I leveraged React Native's built-in asynchronous features, such as async/await syntax and the use of promises, to manage asynchronous tasks effectively. By structuring my code to handle asynchronous operations asynchronously and using error handling mechanisms, I ensured that the application remains responsive and robust even during data fetching and processing. Additionally, I adopted a modular and encapsulated approach to state management, using the useState and useEffect hooks provided by React Native. This allowed me to separate concerns and manage state changes in a clear and organized manner, reducing the likelihood of race conditions or data inconsistencies.

Exercise App

The purpose of this project was to create an Exercise application using React. The application includes a menu with four different exercise options to choose from in which the user can either record laps with a timer or increase their rep count.

The specific skills and technologies I utilized were React for building the user interface components, useState and useCallback hooks for managing state and handling user interactions, and CSS for styling the components.

A problem I encountered during development was efficiently managing the state and transitions between different screens of the application. I addressed this issue by carefully designing the state management logic and using conditional rendering to switch between the menu screen and the exercise screen based on user interactions

Quiz App

The purpose of this project was to create a quiz application using React Native. The application allows users to answer a series of questions presented in various formats, including multiple-choice, multiple-answer, and true-false questions.

The specific skills and technologies I utilized were React Native for building the user interface components, useState and useEffect hooks for managing state and handling user interactions, and AsyncStorage for storing user responses locally on the device.

One challenge I encountered during development was accurately tracking and recording user responses for each question. Given the different question types and the need to handle both single and multiple correct answers, ensuring the correctness of user responses and calculating the final score presented a significant challenge. To address this challenge, I implemented a function to calculate whether the user's selected choices matched the correct answers for each question. By carefully comparing the user's selections with the correct answers and considering the question type, I was able to accurately determine the user's performance for each question and calculate the total score

Exercise App Design

The Figma design for the Exercise app includes a main screen with options for Push Ups, Running, and Plank, with an additional option for Exercise Type. Upon selection, users can access Duration or Repetition Exercise screens, featuring intuitive controls and the exercise name. Navigation options allow returning to the main screen. The Exercise Type menu categorizes exercises into Duration, Repetition, Resistance, and Biking, with added screens for Resistance Training and Biking. In Resistance Training, users can adjust rep count and input weight. The Biking Screen allows editing routes, tracking distance, and pausing the timer.

TODO App Design

The Figma design for the TODO app features a modern interface with a login screen for user authentication. Once logged in, users can create tasks with descriptions and deadlines, manage their task list, mark tasks as completed, and delete tasks as needed. The task list displays task descriptions, deadlines, and completion status, while completed tasks are clearly marked and separated from the incompleted ones. The design includes intuitive navigation and clear indications of different app states for a seamless user experience on a prototypical modern smartphone interface.

Flash Card App Design

The Figma design for the Flash Card app presents a user-friendly interface optimized for studying. Users can effortlessly create, edit, and delete flashcards, with each card displaying a prompt and an answer. Navigation between flashcards is smooth, and users can shuffle the order of cards for varied study sessions.
